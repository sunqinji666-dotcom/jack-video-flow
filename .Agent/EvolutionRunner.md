# EvolutionRunner — 自进化技能

---

## 触发条件

- 每次新开Session，系统自动检测信号队列
- `docs/signals/signal-log.md` 中存在"待处理"状态的信号
- Jack主动说"看一下最近的信号"、"进化一下"

---

## 完成标准

### 必达标准
1. 每条待处理信号被分析完毕，判断了消化方向
2. 从信号中抽象出的规则修改建议，一条条列给Jack看
3. 每条建议标注：是新增、替换、补充还是删除
4. 被Jack确认的建议写入了对应的Skill文件
5. 被Jack拒绝的建议标注了拒绝原因和放弃
6. 处理完的信号状态从"待处理"改为对应的"已处理-XX"
7. 处理完后，信号队列状态归零

### 质量标准
8. 消化的结果不是"把信号原样抄成规则"——而是从信号中抽象出可泛化的规则
9. 不重复的建议——如果是旧问题的变体，合并到现有规则而不是新增
10. 检查有没有从来用不上、建议删掉的规则（主动瘦身）

### 验收方式
- Jack逐条确认或拒绝规则修改建议
- 处理后检查信号日志，确认所有"待处理"已处理
- 如果Jack说"这个进化没意义"，标记失败原因，下次改进消化逻辑

---

## 红线（禁止事项）

- 禁止不经Jack确认直接修改任何Skill文件
- 禁止把一条信号消化成多条规则——一条信号最多对应一条规则修改
- 禁止在消化时脱离原始场景做过度抽象
- 禁止把Jack的情绪化表达（"写得什么鬼"）当成规则信号——必须提取其中可操作的问题
- 禁止用进化结果覆盖掉Jack之前明确说"这个不改"的规则
- 禁止删除"从来没用过的规则"时跳过确认——可能只是那个场景还没触发

---

## 消化流程

### Step 1：信号分析
新建一个独立的子Agent（干净脑子），给它的输入：
1. 信号日志全文
2. 当前所有Skill文件
3. Agent.md顶层规则

让它逐条分析：
- 这个信号反映了什么问题？为什么Jack不满意？
- 现有规则能不能覆盖这个问题？如果能，为什么没挡住？
- 如果不能覆盖，哪条规则需要新增/替换/补充？

输出格式：
```
## 信号：[信号标题]
**分析**：[问题的本质]
**判断**：新增规则 / 替换规则 / 补充规则 / 放弃（因为XX原因）
**影响范围**：[哪些Skill会受影响]
**建议的新规则内容**：[具体写出来]
**建议替换/补充的位置**：[哪个Skill，大概什么位置]
```

### Step 2：去重与合并
主Agent拿到子Agent的分析结果后：
1. 检查多条建议是否有重叠——有的话合并
2. 检查新建议是否与现有规则冲突——冲突就标记出来
3. 对每条建议标注优先级（高/中/低）

### Step 3：呈现给Jack
以清单形式呈现：
```
本轮进化消化了 X 条信号，提出 Y 条规则修改建议：

■ 新增：[规则内容] → 写入 [哪个Skill]
  理由：[从哪条信号来的]
  优先级：高/中/低

■ 替换：[旧规则] → [新规则] → 写入 [哪个Skill]
  理由：[从哪条信号来的]
  优先级：高/中/低

■ 补充：[现有规则] 补充 [新内容] → 写入 [哪个Skill]
  理由：[从哪条信号来的]
  优先级：高/中/低

■ 删除：[规则内容] → 从 [哪个Skill] 删除
  理由：[从来没有触发的证据]
  优先级：高/中/低

请逐条确认：改 / 不改 / 改但我说修改
```

### Step 4：执行落实
Jack确认后，逐条写入对应的Skill文件。
写入后更新信号日志：
```
状态：[待处理] → [已处理-新增/替换/补充/放弃]
处理时间：YYYY-MM-DD
处理结果：[简述]
```

### Step 5：瘦身检查（可选，每5次进化做一次）
主Agent检查所有Skill中的规则：
- 哪些规则从来没用过（没有信号触发过它、没有Jack确认过它的场景）？
- 哪些规则描述的是同一件事但写了两遍？
- 提出删除/合并建议，让Jack确认。

---

## 输出产物
1. 更新后的信号日志（`docs/signals/signal-log.md`）
2. 更新后的Skill文件（被修改的那个）
3. 一份进化摘要：`docs/signals/evolution-log.md`，记录每次进化的改动

---

## 执行约束
- 子Agent消化时，上下文整份带走（信号日志+所有Skill+Agent.md）
- 子Agent返回结果后，主Agent必须做一次"合理性质疑"再呈现给Jack——子Agent也可能偏
- 如果Jack连续两次拒绝同类型的进化建议，把"这类建议不需要"也当成一条信号抓下来
