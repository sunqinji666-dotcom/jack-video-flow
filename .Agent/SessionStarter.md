# SessionStarter — 会话启动协议

---

## 触发条件

每次新开Session，系统自动执行。不需要Jack触发。

---

## 完成标准

### 必达标准
1. 扫描了 `docs/` 下所有文档，判断了当前项目状态
2. 检查了信号队列是否有"待处理"条目
3. 用一句话报告了当前状态
4. 给出了下一步建议

---

## 启动流程

### Step 1：环境扫描

顺序检查以下内容：

**1a. 检查系统配置**
- `Agent.md` 是否存在？不存在→系统未初始化，引导Jack完成初始化
- `.Agent/` 下所有Skill文件是否齐全？不全→列出缺失项

**1b. 检查风格DNA**
- `docs/style-dna/jack-style.md` 是否存在？
- 不存在→标记为优先事项：必须先跑StyleExtractor

**1c. 检查当前项目进度**
按优先级扫描：
- `docs/topics/` → 选题情况（多少个待审核、多少个已通过）
- `docs/research/` → 研究报告情况（哪个选题的研究在进行中或已完成）
- `docs/outlines/` → 大纲情况（哪个选题的大纲在等待确认）
- `docs/scripts/` → 成稿情况（哪个选题的成稿在进行中或已完成）
- `docs/signals/signal-log.md` → 信号队列（多少条待处理）

**1d. 判断当前环节**
根据文档存在情况，判断系统当前处于哪个环节：
- 无选题卡片 → 环节1：选题收集
- 有已通过选题但无研究报告 → 环节2：深度研究（或跳过直接到大纲）
- 有研究报告但无大纲 → 环节3：大纲撰写
- 有未锁定大纲 → 环节3：大纲待确认
- 有锁定大纲但无成稿 → 环节5：成稿+分镜
- 有成稿 → 当前选题已完成，可以开始新选题

### Step 2：信号处理

如果 `docs/signals/signal-log.md` 存在且有待处理信号：
- 自动启动EvolutionRunner，派子Agent消化
- 呈现消化结果给Jack确认

如果信号队列为空：
- 报告"信号队列空，无需进化"

### Step 3：状态报告

用下面的格式报告（必须简洁，一段话）：

```
系统状态：[当前环节名称]
当前选题：[选题标题，如无则"无"]
已完成：[已完成的环节]
下一步建议：[一个具体的动作]
信号状态：[N条待处理 / 空]

需要我的建议？
```

示例：
```
系统状态：大纲撰写
当前选题："AI视频创作工具深度测评"
已完成：选题卡片 ✓ | 研究报告 ✓ | 大纲待确认
下一步建议：打开 docs/outlines/2026-06-14-AI视频创作工具深度测评-大纲.md，确认大纲
信号状态：空

需要我做什么？
```

### Step 4：等待Jack指令

不主动做事。等Jack说下一步。Jack可能：
- 说"继续" → 按当前环节继续执行
- 说"进Evolution" → 手动触发自进化
- 说"开始新选题" → 触发TopicHunter
- 说"看看大纲" → 打开当前大纲让Jack看

---

## 启动快捷操作

| Jack说 | 含义 |
|--------|------|
| "继续" | 从当前环节接着往下走 |
| "新选题" | 跳过当前选题，启动TopicHunter |
| "进化" | 手动启动EvolutionRunner |
| "状态" | 重新报告当前状态 |
| "下一步" | 执行"下一步建议"中的动作 |

---

## 执行约束
- 状态报告必须简洁，不能超过一段话+一个建议。
- 不要让Jack在每次新Session时被一大堆信息淹没。
- 如果系统还没初始化，先引导初始化，不要强行进入选题。
