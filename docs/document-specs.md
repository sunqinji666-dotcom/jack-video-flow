# 文档规范（Document Specs）

> 每个环节产出一份标准格式的MD文档，下一个环节直接消费。
> 这是系统的骨头——格式定清楚了，AI才知道怎么交接。

---

## 一、选题卡片格式

文件命名：`docs/topics/YYYY-MM-DD-选题名.md`

```markdown
# [选题标题]

**采集时间**：YYYY-MM-DD HH:MM
**来源**：[热点追踪 / 社交媒体 / 项目经验库 / Jack指定]
**视频类型**：[知识科普 / 项目复盘 / 个人Vlog]
**难度**：[轻 / 中 / 重]
**预期时长**：[X分钟]

## 选题概述
[一两句话说清楚这个选题要讲什么]

## 为什么现在做
[时效性、热点关联、个人经验关联]

## 观众得到什么
[看完这个视频，观众获得什么信息或感受]

## 可能的切入角度
- 角度1：[简述]
- 角度2：[简述]
- 角度3：[简述]

## Jack的独特优势
[为什么Jack比其他人更适合讲这个选题]

## 是否需要深度研究
[是/否 — 知识科普类默认需要]

## 状态
[待审核 / 已通过 / 已放弃]
```

## 二、研究报告格式

文件命名：`docs/research/YYYY-MM-DD-选题名.md`

```markdown
# [选题标题] — 深度研究报告

**研究时间**：YYYY-MM-DD
**信息源数量**：[X]
**研究深度**：[深度 / 中度]

---

## 一、核心事实
[这个选题最核心的客观事实是什么。用简洁的条目式，每一条附带来源标注]

## 二、关键数据
[所有可引用的数据，标注来源、时间、数据口径]

## 三、不同观点
[这个选题上存在的分歧。列出不同立场及其代表观点，标注来源]

## 四、有趣的细节
[能转化成画面钩子的细节、故事、人物、场景]

## 五、参考资料列表
- [来源1]：链接/出处，可靠度评估
- [来源2]：链接/出处，可靠度评估
- ...

## 六、AI使用说明
[标注这份报告哪个部分是给AI看的，AI写文案时应该重点参考哪些]
```

## 三、视频大纲格式

文件命名：`docs/outlines/YYYY-MM-DD-选题名-大纲.md`

```markdown
# [选题标题] — 视频大纲

**大纲日期**：YYYY-MM-DD
**预期视频时长**：[X分钟]
**视频类型**：[知识科普 / 项目复盘 / 个人Vlog]

---

## 备选标题
1. [标题方案1] — [为什么这个标题好]
2. [标题方案2]
3. [标题方案3]

## 开场钩子（前15-30秒）
[用什么画面+什么话把人抓住]
**情绪定位**：[好奇 / 惊讶 / 共鸣 / 悬念]

## 段落结构

### 段落1：[段落标题]
- **核心信息**：[想传递什么]
- **情绪定位**：[这一段观众应该是什么感受]
- **画面方向**：[大概什么画面]
- **时长预估**：[X秒/分钟]
- **参考素材**：[已有素材 or 需要拍摄 or 需要找]

### 段落2：[段落标题]
[同上]

### 段落N：[段落标题]
[同上]

## 情绪曲线
[文本描述整条视频的情绪起伏路线]

## 结尾设计
[怎么收尾——回钩开场 / 金句 / 行动号召 / 留白]

## 画面视觉建议
[整体色调、节奏、关键视觉元素]
```

## 五、成稿+分镜格式

文件命名：`docs/scripts/YYYY-MM-DD-选题名-成稿.md`

```markdown
# [选题标题] — 完整文案与分镜

**成稿日期**：YYYY-MM-DD
**预期视频时长**：[X分钟]

---

## 画面说（整体视觉方向）
[3-5句话描述整个视频的视觉气质和节奏]

---

| 序号 | 画面说明 | 口播内容 | 情绪 | 时长(秒) | 镜头建议 | B-roll方向 |
|------|---------|---------|------|---------|---------|-----------|
| 1 | [画面描述] | [说的内容] | [情绪标注] | [X] | [镜头类型] | [素材方向] |
| 2 | ... | ... | ... | ... | ... | ... |

---

## 转场提示
| 从 | 到 | 转场方式 | 节奏 |
|----|-----|---------|------|
| 段落1 | 段落2 | [硬切/叠化/黑场/声音衔接] | [快/慢/停顿] |

---

## 素材清单
- [ ] 需要拍摄：[列出需要拍的画面]
- [ ] 需要找：[列出需要找的素材]
- [ ] 已有素材：[列出可以直接用的已有素材]
```

## 六、信号日志格式

文件命名：`docs/signals/signal-log.md`

```markdown
# 信号日志

> 每次Jack纠正/质疑/强调时自动记录。新Session时由EvolutionRunner消化成规则建议。

---

## [YYYY-MM-DD HH:MM] [信号标题]

**触发环节**：[选题/研究/大纲/风格/成稿]

**触发场景**：
[在什么环节、做什么操作时触发的]

**Jack原话（或行为描述）**：
[Jack说了什么，做了什么]

**AI原本的做法**：
[AI当时做了什么让Jack不满意]

**问题本质**：
[为什么这是问题，反映了什么不足]
[优先级：高/中/低]

**状态**：[待处理 / 已处理-新增 / 已处理-替换 / 已处理-放弃]

---
```
